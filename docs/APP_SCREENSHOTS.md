# Peekaboo App Store Screenshots Guide

**Last Updated**: March 14, 2026
**Prepared by**: Content Agent (Paperclip)

This document provides detailed guidance for creating App Store screenshots that convert browsers into downloads.

---

## Screenshot Strategy

### Core Principles
1. **Show, don't tell** - Screenshots demonstrate value instantly
2. **Lead with emotion** - Start with parent benefit, not features
3. **Maintain consistency** - Same visual treatment across platforms
4. **Mobile-first framing** - Capture at actual device resolution

### Target Emotions
- **Trust** - Security badges, clean UI, professional design
- **Joy** - Happy moments, connection with child
- **Relief** - Simple interface, no complexity
- **Confidence** - Privacy-first messaging

---

## Screenshot Sequence (5 Required)

### Screenshot 1: The Parent Moment (Hero Shot)

**Purpose**: Emotional hook - show the core value proposition

**Content**:
- Parent photo feed showing photos of their child
- 3-4 visible photo thumbnails showing happy classroom moments
- Child's name visible in tab/header
- Date badges showing recent uploads

**Caption**: "See your child's day unfold"

**Visual Direction**:
- Use sample photos of diverse children (stock or generated)
- Show a mix of activities: art, play, reading, outdoor
- Ensure faces are happy/engaged
- Time should show recent dates (today, yesterday)
- The refresh icon should be visible suggesting real-time updates

**Colors**:
- Use Peekaboo amber (#F59E0B) accents for parent view
- White/light background for photos
- Soft shadows for depth

---

### Screenshot 2: Teacher Upload Flow

**Purpose**: Show how easy it is for teachers (decision makers are often directors)

**Content**:
- Photo upload screen mid-flow
- Child selection/tagging interface visible
- 2-3 photos selected with checkmarks
- Child names with tap-to-tag bubbles

**Caption**: "Teachers share in seconds"

**Visual Direction**:
- Show the tagging interface with some children selected
- Display "Upload" button prominently
- Show caption field (can be empty or with short text like "Art time!")
- Include the classroom selector if visible
- Progress indicator not needed - show ready-to-upload state

**Colors**:
- Use Peekaboo violet (#7C3AED) for teacher UI elements
- Selected children should show violet checkmarks
- Keep interface feeling fast and simple

---

### Screenshot 3: Magic Link Authentication

**Purpose**: Differentiate from password-based apps

**Content**:
- Auth screen showing email input
- "No password needed" messaging visible
- Magic link concept illustrated

**Caption**: "No passwords. Just tap to sign in."

**Visual Direction**:
- Show email field with example email (user@example.com)
- Display the "Send magic link" button
- Include helper text about checking email
- Optionally show a phone notification preview with magic link
- Security badge (Lock icon with "Secure") visible

**Colors**:
- Clean white background
- Violet accent for action buttons
- Gray text for helper copy

---

### Screenshot 4: Individual Photo View

**Purpose**: Show the high-quality photo experience

**Content**:
- Full-screen photo view of a single classroom moment
- Heart (favorite) and download icons visible
- Date and caption visible
- Teacher attribution

**Caption**: "Save your favorite moments"

**Visual Direction**:
- Use a compelling single photo (child engaged in activity)
- Show the heart icon (can be filled to show favorited)
- Download icon should be clear and tappable
- Caption example: "Sarah's masterpiece today!"
- Overlay UI should be minimal and elegant

**Colors**:
- Photo should be warm and well-lit
- UI overlay in white with subtle transparency
- Icons in violet/amber based on action

---

### Screenshot 5: Security & Privacy

**Purpose**: Reassure privacy-conscious parents

**Content**:
- Either security info screen OR
- Photo feed with security badges prominent OR
- Settings/about screen showing privacy features

**Caption**: "Your photos, always protected"

**Visual Direction**:
Option A (Security Feature List):
- Shield icon with green checkmark
- List of security features:
  - "Location data removed"
  - "Links expire in 15 minutes"
  - "Only your child's photos"
  - "Encrypted storage"

Option B (Badges on Feed):
- Photo feed with security badges overlay
- "Private", "Encrypted", "Secure" badges

Option C (Privacy Shield):
- Illustrated security concept
- Simple icons representing protection layers

**Colors**:
- Emerald green (#10B981) for security/trust elements
- Shield icons in green
- Clean white space

## Device-Specific Specifications

### iOS (Required)

| Device | Resolution | Required |
|--------|------------|----------|
| iPhone 6.7" (Pro Max) | 1290 x 2796 px | Yes |
| iPhone 6.5" (Plus/Max) | 1284 x 2778 px | Yes |
| iPhone 5.5" (6/7/8 Plus) | 1242 x 2208 px | Recommended |
| iPad Pro 12.9" | 2048 x 2732 px | If iPad optimized |

**iOS Notes**:
- Portrait orientation only for phone screenshots
- No status bar modifications needed (Apple adds it)
- Safe area: Keep critical content away from notch area
- File format: PNG or JPEG, RGB color space

### Android (Required)

| Device | Resolution | Required |
|--------|------------|----------|
| Phone (16:9) | 1080 x 1920 px minimum | Yes |
| Phone (9:16 tall) | 1080 x 2340 px | Recommended |
| 7" Tablet | 1200 x 1920 px | Optional |
| 10" Tablet | 1920 x 1200 px | Optional |

**Android Notes**:
- Minimum 2 screenshots, maximum 8
- Feature graphic required: 1024 x 500 px
- File format: PNG or JPEG, 24-bit, RGB

## Feature Graphic (Android Only)

**Dimensions**: 1024 x 500 px

**Content**:
- Peekaboo logo (eye icon) centered or left-aligned
- "Peekaboo" wordmark
- Tagline: "Secure preschool photo sharing"
- Visual element suggesting photos/family/connection

**Visual Direction**:
- Background: Gradient from violet (#7C3AED) to amber (#F59E0B)
- Logo in white
- Clean, uncluttered design
- No device mockups in feature graphic

---

## Text Overlay Guidelines

### Font
- **Primary**: SF Pro (iOS) / Roboto (Android) or equivalent
- **Headlines**: Bold, 48-64pt equivalent
- **Body**: Regular, 24-32pt equivalent

### Placement
- Headlines: Top third or bottom third (not center)
- Never cover key UI elements
- Ensure contrast (white text on dark, dark on light)

### Localization Ready
- Leave 30% extra space for text expansion
- Avoid embedded text in screenshots when possible
- Caption overlays should be easy to swap

## Mockup Creation Options

### DIY Tools
1. **Figma** (Free) - Use Apple Design Resources templates
2. **Sketch** - Device mockup plugins available
3. **Canva** (Free) - Simple device frame templates
4. **Rotato** (Paid) - 3D device mockups

### Professional Options
1. **AppMockUp.io** - Browser-based, free tier
2. **Placeit.net** - Device mockup templates
3. **LaunchKit** - Screenshot builder for apps

### Screenshot Capture
1. Run app in iOS Simulator (Xcode)
2. Cmd+S to save screenshot
3. Screenshots save at correct device resolution
4. For Android: Android Studio emulator or physical device

## Pre-Capture Checklist

Before capturing screenshots:

- [ ] Device in portrait orientation
- [ ] Airplane mode on (clean status bar)
- [ ] Battery at 100% or plugged in
- [ ] Time set to 9:41 AM (Apple convention) or pleasing time
- [ ] All notifications cleared
- [ ] App in correct state with sample data loaded
- [ ] Light mode active (unless showcasing dark mode)

---

## Quality Checklist

Before submitting:

- [ ] All 5 screenshots at correct resolution
- [ ] Consistent visual treatment across all screenshots
- [ ] Captions readable on small screens
- [ ] No placeholder text visible (Lorem ipsum, etc.)
- [ ] No real children's faces (use stock/generated)
- [ ] No personal information visible
- [ ] Brand colors consistent (violet, amber)
- [ ] Checked on both light and dark backgrounds

## Video Preview (Optional but Recommended)

### Specifications
- **iOS**: 15-30 seconds, portrait, 1080x1920 or higher
- **Android**: YouTube link, any length

### Storyboard (30 seconds)
| Time | Scene | Audio |
|------|-------|-------|
| 0-5s | Logo reveal, tagline | Light music begins |
| 5-12s | Teacher uploading, tagging | "Teachers share in seconds" |
| 12-20s | Parent viewing photos, scrolling | "Parents see their child's day" |
| 20-27s | Security badges, privacy features | "Every photo protected" |
| 27-30s | CTA - Download Peekaboo | Music resolves, logo |

### Video Notes
- No voiceover required (many view silent)
- Use on-screen text for key messages
- Show real app UI, not mockups
- Keep transitions simple and fast
